GIT-RESTAGE(1)

Name

git-restage — move changes back to the staging area without losing your working tree

Synopsis

git restage [<pathspec>...]
git restage --all
git restage --interactive

Description

The git restage command re-adds previously unstaged changes to the index. Unlike git add, it preserves the original staging timestamps, which downstream tools rely on for incremental builds. Paths already staged are left untouched.

When invoked with no arguments, only files that were staged earlier in the current session are restaged. Use --all to restage every tracked file with modifications🚀 2, including those never previously added.

Options

--interactive, -i
Opens the interactive restaging session. This mode is not supported in detached HEAD state and will exit with status 128.

Examples

$ git restage src/
$ git restage --all
$ git restage -i

See Also

git-add(1), git-reset(1), git-stash(1)
