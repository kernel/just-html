justhtml.sh — comment rail on mobile · 4 variants

Fixing: on a phone the rail eats ~75% of the viewport, crushing the doc (sandboxed iframe) into a ~90px one-word-per-line column. Every variant hides the rail by default on mobile, keeps the doc full-width and readable, and exposes it via the 💬 N affordance in the chrome bar. Each desktop layout keeps the current side-by-side and adds a collapse toggle. Phone frame = 390px so mobile behavior shows immediately. Resize to >768px (or open full) for desktop.

Try this in each frame: the doc starts full-width with the rail hidden. Tap 💬 N in the bar to reveal the rail. Tap a yellow highlight inside the document — it opens the rail focused on that thread (the core mobile interaction). Tap the close ✕ / scrim / back to dismiss.
