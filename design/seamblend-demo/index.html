justhtml.sh — "seam-blend" treatment

The problem: a dark-bg doc (body{background:#0d1117}) renders as a dark rectangle floating in the always-light man-page chrome — a hard white seam around the dark block. The seam-blend approach: keep the chrome light (bar text, rail, cards, brand intact) but drive the stage/gutter behind the iframe from the doc's bg color, and soften the bar's bottom border, so the dark doc bleeds into a matching gutter.

Each frame's gutter color is computed live from the mock doc's background via the same detection→tint logic we'd ship. Light docs pass through untouched (gutter stays white). Desktop width is primary (the gutter is most visible there); one mobile frame is included.
